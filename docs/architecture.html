SYS // ARCHITECTURE BRIEF

HELM

A voice-controlled heads-up display for your own life — local-first, file-backed, with a background agent doing the heavy thinking. Everything below runs on your machine; the cloud appears exactly twice, and both times are optional or swappable.

01 The map

Six processes, one folder of plain files in the middle. The vault is the shared memory — every component reads and writes markdown, JSON, and CSV. No database, no message bus. If you can read a folder, you can read the whole system's state.

02 Voice round trip

Hold space, talk, release. Speech never leaves your machine — transcription and synthesis are both local. Measured on a consumer GPU:

HOLD SPACEmic → webm clip
TRANSCRIBEfaster-whisper, local100–150ms
ROUTErules engine~0ms
REPLYcomposed from vault state~25ms
SPEAKKokoro TTS, local GPUfirst audio ~270ms

release → reply: 175–500ms speech starts < 1s zero network

03 The router — three engines, three tiers

Every utterance is classified by a chain of engines, cheapest first. Rules answer only the clear-cut (exact rundown phrases, bare skill names, dashboard metrics); anything sentence-shaped goes to a model that can actually read.

RULESregex · ~0ms · clear-cut only → HAIKUAPI · ~1–2s · if keyed → LOCAL MODELOllama · ~600ms · offline → RULES FLOORnever dies
T1

Dispatch

Run a skill. Writes an intent file to the queue, acks instantly, runner takes it from there.

run the inbox audit
T2

Instant answer

Answered from the vault snapshot in ~25ms — metrics, schedule, priorities, the spoken daily rundown.

how many subscribers do I have?
T3

Background ask

Needs real thinking. Becomes a headless Claude run; the answer is spoken when it lands (30s–2min).

draft a reply to that sponsor email

04 Skill lifecycle — buttons are real

Every skill — voice or button — moves through the same file-based pipeline. Each arrow is a file appearing on disk, which means every step is inspectable after the fact.

Intent lands in the queuea tiny JSON file — skill name, label, timestamp
vault/system/queue/<id>.json
Runner claims itdaemon polls every 1.5s; max 3 concurrent; dedupes repeats
runner-status.json heartbeat
Headless Claude does the workclaude -p with the skill's prompt and an explicit model — reads your inbox, scans trends, drafts replies
claude -p --model …
Results written backdeliverable markdown + a run record; first line of the summary is one spoken-friendly sentence
inbox/…/report.md · system/runs/<id>.json
HUD notices and speakstask callout morphs into the document card; HELM announces the summary; click to read in the overlay
state poll → callout → overlay

Mental model: the voice layer is a dispatcher, not a worker. It acks in milliseconds, hands the job to the queue, and announces when results land. Speech never blocks on agent work.
